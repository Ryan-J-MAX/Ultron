# Dialogue Reading（对话式读书）

通过与AI实时对话的方式高效阅读一本书。核心不是复述书中内容，而是**让书和读者的现实发生化学反应**。

## 触发条件

用户说"来读这本书"、"一起读"、"讨论这本书"、"开始读书"、"继续读XX书"，或直接发送文件路径/书名。

## 核心原则

1. **不代替用户思考** — 提取框架和原文细节，不替用户下结论。选择权在用户手里。
2. **对照现实** — 每章讨论时把书中框架和读者现状做对比。用表格呈现"书中标准 vs 你的现状"，标注 ✅ ⚠️ ❌ 快速识别。可以基于判断说"你已经在做""你目前缺失的是"，但最终结论和选择由用户自己做。
3. **延伸自由** — 讨论可以跳到投资、职业、策略等现实话题，不限于书的内容
4. **自动沉淀** — 讨论结束后生成结构化笔记+完整对话记录+TODO清单
5. **100%尽力** — 通读提取框架时做到完整覆盖，讨论时提供充分的原文细节。读者已明显掌握的章节可以根据对话节奏快速掠过，但不能跳过通读。

## 输入方式

- **方式A（推荐）**：用户提供本地 TXT 文件路径。支持大文件分块（/tmp/book_chunks/）
- **方式B**：用户提供书名+作者，通过搜索获取摘要和目录

## 阅读流程（4步）

### 第1步：建立上下文

1. 确认书名、作者、字数、章节结构
2. 了解读者背景和当前阶段（财务状况、知识储备、相关经验）
3. 告知用户全书规模和阅读计划

### 第2步：逐章推进

每章节奏：
1. **通读原文**（100%尽力，完整覆盖）
2. **提取核心框架**（3-5个要点，不逐字复述）
3. **关键概念/故事**（选1-2个最有冲击力的）
4. **对照读者现实**（最重要一步）
   - 用表格对比"书中标准 vs 你的现状"
   - 标注 ✅ ⚠️ ❌ 快速识别
   - 可以基于判断说"你已经在做""你目前缺失的是这个"
   - 最终结论和选择由用户自己做
5. **练习/填空**（如果书中有交互练习）
   - 等用户输入
   - 不要替用户填
6. 如果读者已明显掌握某章内容，可根据对话节奏快速掠过（但仍需通读原文）

### 第3步：讨论延伸

- 允许读者偏离书中内容，讨论自己的真实困惑
- 用书中框架回答读者的问题
- 延伸相关知识点和跨书对比

### 第4步：收尾

1. **结构化笔记** → `book-notes/书名/notes-YYYY-MM-DD.md`
   - 每章核心框架 + 读者对照 + 关键讨论
2. **完整对话记录** → 使用 `export-session` skill 导出逐字原文
   - 执行 `openclaw sessions export-trajectory --session-key "agent:main:main" --workspace . --json`
   - 用 Python 解析 `events.jsonl` 提取 user/assistant 消息 → `full-dialog.md`
   - 如有前半段在飞书等外部平台的对话，单独导出后合并
   - 最终文件：`book-notes/书名/chat-YYYY-MM-DD.md`
3. **TODO清单** → 书中建议的行动项，按优先级排列，可操作
4. **清理** → 删除临时 trajectory export 目录

## 输出风格

- **直接，不废话**。不写"这是一个很好的问题"，直接回答
- **对比表格优先**。对照读者现状时用表格，一目了然
- **说"你"不是说"读者"**。这是对话，不是论文
- **控制每章篇幅**。一章不超过一屏，核心要点+对照即可

## 文件规范

```
book-notes/
├── README.md                      ← 书单索引
└── 书名/
    ├── README.md                  ← 书的基本信息 + 一句话总结
    ├── notes-YYYY-MM-DD.md        ← 结构化笔记
    └── chat-YYYY-MM-DD.md         ← 完整对话记录（export-session导出）
```

## 已验证案例

- 《以日为鉴》（36.8万字，5篇15章，约4小时）
- 《财务自由之路》（43.5万字，14章+自我分析，约3小时）
